# แบบฝึกหัด EP4 — แผนที่ข้อมูลเซนเซอร์ IoT

## ส่วนที่ 1: อ่าน Contour และตรวจสอบจุดร้อน

ใช้ `image_and_contour.py` เป็นจุดเริ่มต้น:

1. ระบุความหมายของเส้น 24, 26, 28, 30 และ 32 °C
2. อธิบายว่าบริเวณใดมีอุณหภูมิเปลี่ยนเร็ว โดยดูจากระยะห่างของเส้น
3. ย้ายเครื่องจักรไปที่ (6, 5) แล้วสังเกตรูปร่างของเส้น
4. ย้ายเครื่องปรับอากาศไปที่ (1, 1)
5. ทำเส้นเตือนภัย 30 °C ให้เป็นสีแดงและหนากว่าเส้นอื่น

## ส่วนที่ 2: จำลอง Measurement Error

1. เปลี่ยนค่า standard deviation ของ noise เป็น 0.05, 0.15 และ 0.50 °C
2. เปรียบเทียบค่าจริงกับค่าที่อุปกรณ์รายงาน
3. เพิ่ม bias คงที่ +0.4 °C ให้เซนเซอร์ T3
4. จำลอง drift ของ T8 ที่เพิ่มขึ้นตามเวลา
5. อธิบายความแตกต่างระหว่าง noise, bias และ drift

## ส่วนที่ 3: ทดลอง Interpolation

ใช้ `sensor_interpolation.py`:

1. สร้างแผนที่ด้วย `nearest`, `linear` และ `cubic`
2. เปรียบเทียบพื้นที่ที่คำนวณไม่ได้ของแต่ละวิธี
3. คำนวณ MAE ของแต่ละวิธีจากข้อมูลจำลอง
4. ลบ sensor node ใกล้เครื่องจักรออก แล้วดูว่า hot spot เปลี่ยนอย่างไร
5. เพิ่ม sensor node ใกล้จุดร้อนอีกสองตัว แล้วคำนวณ MAE ใหม่
6. อธิบายว่าทำไมภาพที่เรียบกว่าอาจไม่ได้แม่นยำกว่า
7. ระบุพื้นที่ซึ่งเป็น Interpolation และพื้นที่ซึ่งจะกลายเป็น Extrapolation

## ส่วนที่ 4: จำลองปัญหาของระบบจริง

1. กำหนดให้ T3 และ T8 offline
2. เก็บค่าที่หายไปเป็น `NaN`
3. แสดงอุปกรณ์ offline เป็นกากบาทสีเทา
4. เพิ่ม timestamp และข้อความแสดงเวลาอัปเดตล่าสุด
5. ปฏิเสธค่าจากอุปกรณ์ที่มี timestamp เก่าเกินกำหนด
6. เปรียบเทียบช่วงสีอัตโนมัติกับช่วงคงที่ 22–34 °C

## ส่วนที่ 5: เปลี่ยนชนิดของเซนเซอร์

เลือกหนึ่งตัวแปร:

- ความชื้นสัมพัทธ์ (%RH)
- PM2.5 (µg/m³)
- CO₂ (ppm)
- ความชื้นดิน (%)
- RSSI (dBm)
- แรงสั่นสะเทือน RMS (mm/s หรือ g)

สร้างข้อมูล เลือก colormap กำหนด colorbar และ threshold ให้สอดคล้องกับความหมายของตัวแปร

## โจทย์ท้าทาย: IoT Spatial Dashboard

สร้าง Dashboard ที่มี:

- แผนที่สีและเส้น threshold
- ตำแหน่ง รหัส และค่าของอุปกรณ์
- หน่วยและ Colorbar
- สถานะ online/offline
- เวลาอัปเดตล่าสุด
- ช่วงสีคงที่สำหรับเปรียบเทียบต่างเวลา
- คำอธิบายว่าบริเวณใดเป็นค่าที่วัดจริง ค่า Interpolation หรือ Extrapolation
- ตัวชี้วัดความคลาดเคลื่อนของแบบจำลอง
