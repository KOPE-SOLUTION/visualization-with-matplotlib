# แบบฝึกหัด EP5 — Histograms, Binning, and Density for IoT

## ส่วนที่ 1: อุณหภูมิห้องเครื่องจักร

1. สร้างข้อมูลอุณหภูมิปกติ 2,000 ค่า
2. เพิ่มข้อมูล overheat อีก 100 ค่าที่มีค่าเฉลี่ย 35 °C
3. เปรียบเทียบ histogram ที่ใช้ 15, 40 และ 100 bins
4. อธิบายว่าจำนวน bin ใดมองเห็น overheat ได้ชัดที่สุด
5. คำนวณจำนวนและสัดส่วนข้อมูลที่สูงกว่า 32 °C

## ส่วนที่ 2: เปรียบเทียบเครื่องจักร

จำลองข้อมูล vibration ของมอเตอร์สามตัว แล้วสร้าง histogram ซ้อนกัน โดยต้องมี:

- สีที่แตกต่างกัน
- `density=True`
- ความโปร่งใส
- ชื่อแกนและหน่วย
- Legend
- เส้น threshold ที่กำหนดโดยผู้ดูแลระบบ

## ส่วนที่ 3: Edge Gateway Summary

สร้าง Histogram Summary ที่ใช้ Fixed Bin Edges และมี Device ID, ช่วงเวลา, หน่วย, Sampling Rate, Bin configuration version, Bin edges และ Counts จากนั้นอธิบายว่าทำไม Summary จากคนละ Bin configuration จึงนำมารวมกันไม่ได้

## ส่วนที่ 4: ข้อมูลสองมิติ

เลือกหนึ่งคู่ตัวแปร:

- กระแสไฟกับอุณหภูมิมอเตอร์
- อุณหภูมิกับความชื้น
- ความเร็วรอบกับแรงสั่นสะเทือน
- RSSI กับระยะทางจาก Gateway

สร้างทั้ง 2D histogram และ hexbin จากข้อมูลเดียวกัน แล้วเปรียบเทียบว่ากราฟใดอ่าน operating region ได้ชัดกว่า

## ส่วนที่ 5: KDE

สร้าง histogram พร้อม KDE จากข้อมูลเซนเซอร์หนึ่งตัวแปร แล้วทดลองปรับ bandwidth ให้เส้น:

- ละเอียดมาก
- สมดุล
- เรียบมาก

อธิบายว่าแต่ละแบบอาจทำให้ผู้ดูแลระบบตีความผิดอย่างไร

## โจทย์ท้าทาย: Predictive Maintenance Summary

สร้างรายงานที่ประกอบด้วย:

- Histogram เปรียบเทียบสถานะปกติและสถานะเตือน
- ค่าเฉลี่ย ส่วนเบี่ยงเบนมาตรฐาน และ percentile ที่ 95
- เส้น threshold
- จำนวนค่าที่เกิน threshold
- ข้อสรุปว่าควรตรวจสอบเครื่องจักรหรือไม่
- คำเตือนว่า histogram ไม่แสดงลำดับเวลา
